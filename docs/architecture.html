max-agent

Pattern de deliberare multi-perspectivă pentru evaluarea schimbărilor de cod
Constitution — guvernează fiecare deliberare
1. Think before coding — expune tradeoff-uri, nu presupune.
2. Simplicity first — minimum de cod care rezolvă problema.
3. Surgical changes — atinge doar ce cere goal-ul.
4. Goal-driven — success_criterion + verification obligatorii.

Cele 3 voci

Generator

Creativ
Produce 3-5 abordări candidate prin gândire divergentă. Include mereu do_nothing ca baseline.
  • Curios, nu prudent — riscul e jobul altcuiva
  • Cantitate înainte de calitate
  • Fără auto-cenzură
Nu are voie: să filtreze pentru risc sau fezabilitate.

Control

Analitic
Verifică tehnic fiecare candidate: types, logică, tests, style. Marchează valid: true/false.
  • Pedant, nu pesimist
  • Concret peste abstract
  • Verifică, nu speculează
Nu are voie: să respingă pentru risc — pass through dacă e corect.

Conservator

Sceptic
Scorează risc pentru fiecare candidate valid: diff_size, scope_drift, regression_risk, reversibility.
  • Reversibilitatea peste deșteptăciune
  • Blast radius contează
  • Poate recomanda do_nothing
Nu are voie: să re-valideze correctness — trust Control.

Aggregation — 4 scheme de votare

majority Media voturilor; tiebreak pe varianță minimă.
weighted Medie ponderată cu weights ∈ [0.2, 0.49], sum=1.0.
conservative_override (default) Medie ponderată cu safety = 1 − conservator; Conservator are veto unilateral. VETO la risk_score > 0.7
risk_adjusted_utility utility = mean(gen, ctrl, 1−cons) cu penalty sigmoidal centrat la risc=0.5. Fără veto rigid — un candidate cu risc 0.7 e penalizat ~21%, nu disqualified. Folosește când mai mulți candidates sunt strânși în jurul pragului.
Separation of powers: fiecare voce e împiedicată prin prompt să facă jobul celorlalți. Constitution bate orice voce în caz de conflict. Dacă toți candidates sunt vetoiți → aggregator-ul atașează un bloc retry_suggested cu pragul minim sub care candidatul cu risc minim ar fi supraviețuit (capped la 0.85). Decizia (re-roll cu constraint mai strict, accept chosen: null, sau întreabă user-ul) revine agentului principal.

Flow models — 5 moduri (4 active + 1 SPEC)

Sequential — blind DEFAULT

Un context, dar fiecare voce primește doar strictul necesar.
turn 1 — context unic
Claude main
↓ joacă voci, cu strip între
Generator
strip → {id, summary, sketch}
Control
strip → {id, summary, sketch} pt valid
Conservator
✓ Contaminare redusă fără overhead parallel — Control nu vede rationale, Conservator nu vede issues/notes.
Use when: majoritatea deliberărilor — rapid, dar cu Chinese wall implementat prin scripts/strip_context.py.

Parallel voices OPT-IN

Până la 3 sub-agenți Claude independenți, 1 + 2 pattern.
turn 1 — 1 sub-agent
Claude main
↓ Agent call
Generator (sub-agent)
turn 2 — 2 sub-agenți în paralel
Claude main
↓ 2 Agent calls în același message
Control (sub-agent)
Conservator (sub-agent)
✓ Independență reală — sub-agenții returnează doar JSON final, fără reasoning comun.
✓ Telemetry capturat de orchestrator per voce: tokens_in/out ≈ len(prompt|response)/4, latency_ms = wall-clock per Agent call. Best-effort, dar non-zero — altfel runs paralele apar ca 0 tokens în usage.py.
Use when: decizii subtile unde contaminarea contează, high-stakes (migrări DB, security, refactor mare).

Dialectic — two-pass OPT-IN

Două runde paralele cu revizuire cross-context controlată.
pass 1 — independent (ca parallel)
Claude main
↓ 1 + 2 pattern
Generator Control Conservator
pass 2 — revision round (3 paraleli)
Claude main
↓ fiecare voce vede outputs-urile celorlalți
Gen revised
Ctrl revised
Cons revised
✓ Independență în pass 1 + șansă de update în pass 2. revision_log auditează ce s-a schimbat. Merge via scripts/dialectic_merge.py.
Use when: decizii subtile unde o voce ar putea schimba opinia dacă vede ce-au zis celelalte. Cost: 2× parallel (6 sub-agenți total).

Dialectic — iterative (1-3 runde) SPEC

Extensie a dialectic two-pass cu N revision rounds și convergence detection.
pass 1 — independent (ca parallel)
Claude main
↓ 1 + 2 pattern
Generator Control Conservator
pass 2 … pass N+1 — revision loop (N ∈ {1,2,3})
Claude main
↓ fiecare voce vede outputs-urile celorlalți din round-ul anterior
Gen rev.
Ctrl rev.
Cons rev.
✓ Convergence stop — early-exit dacă în round-ul curent: (a) lista de candidate id-uri identică, (b) niciun valid n-a flipat, (c) niciun risk_score n-a variat cu >0.05. Hard cap N=3.
⚠ Nu e încă implementat — dialectic_merge.py trebuie să accepte {rounds: [pass1, pass2, ...]} și să returneze revision_log per round. Schema veche {pass1, pass2} rămâne backward-compat.
Use when: decizii high-stakes unde Conservator-ul flag-uiește un risc neașteptat și vrei să dai Generator-ului șansa de a propune candidate ce evită explicit acel risc — apoi Conservator-ul șansa de a rescore. Cost: 1 + 3·(N+1) sub-agenți (4 / 7 / 10 pentru N = 0 / 1 / 2 / 3).

Sequential — naive LEGACY

Original mode — păstrat pentru referință.
turn 1 — context partajat
Claude main
↓ joacă pe rând, full context
Generator Control Conservator
⚠ Cross-contamination — Control vede rationale, Conservator vede tot ce a spus Control.
Use when: debugging skill-ului însuși, când vrei să vezi raționamentul intermediar — nu pentru deliberări reale.

Cum circulă informația între voci

contaminated path (rationale/issues leak)
stripped path (minimum fields only)
independent sub-agent (true isolation)
controlled cross-context (dialectic pass 2)
convergence loop back (iterative — early-stop)
Max 3 sub-agenți activi simultan; în practică 1 + 2 pentru că Control și Conservator depind de output-ul Generator-ului. Prompts-urile prompts/*.md sunt deja self-contained — pot fi date direct unui sub-agent fără ajustări.
