max-agent

Pattern de deliberare multi-perspectivă pentru evaluarea schimbărilor de cod
Constitution — guvernează fiecare deliberare
1. Think before coding — expune tradeoff-uri, nu presupune.
2. Simplicity first — minimum de cod care rezolvă problema.
3. Surgical changes — atinge doar ce cere goal-ul.
4. Goal-driven — success_criterion + verification obligatorii.

Cele 3 voci

Generator

Creativ
Produce 3-5 abordări candidate prin gândire divergentă. Include mereu do_nothing ca baseline.
  • Curios, nu prudent — riscul e jobul altcuiva
  • Cantitate înainte de calitate
  • Fără auto-cenzură
Nu are voie: să filtreze pentru risc sau fezabilitate.

Control

Analitic
Verifică tehnic fiecare candidate: types, logică, tests, style. Marchează valid: true/false.
  • Pedant, nu pesimist
  • Concret peste abstract
  • Verifică, nu speculează
Nu are voie: să respingă pentru risc — pass through dacă e corect.

Conservator

Sceptic
Scorează risc pentru fiecare candidate valid: diff_size, scope_drift, regression_risk, reversibility.
  • Reversibilitatea peste deșteptăciune
  • Blast radius contează
  • Poate recomanda do_nothing
Nu are voie: să re-valideze correctness — trust Control.

Aggregation — 3 scheme de votare

majority Media voturilor; tiebreak pe varianță minimă.
weighted Medie ponderată cu weights ∈ [0.2, 0.49], sum=1.0.
conservative_override (default) Medie ponderată, dar Conservator are veto unilateral. VETO la risk_score > 0.7
Separation of powers: fiecare voce e împiedicată prin prompt să facă jobul celorlalți. Constitution bate orice voce în caz de conflict. Conservator are putere de veto, dar dacă veto-iește tot → chosen: null și raportul declară explicit „all vetoed”.

Sequential vs Parallel voices

Sequential (default)

Un singur context joacă toate 3 vocile pe rând.
turn 1 — singur, dintr-un context
Claude main
↓ joacă pe rând
Generator Control Conservator
⚠ Risc de cross-contamination — Generator-ul „știe” deja ce va spune Control.
Use when: deliberări mici/rapide, schimbări izolate, când overhead-ul a 3 sub-agenți nu se justifică.

Parallel voices (opt-in)

Până la 3 sub-agenți Claude independenți, 1 + 2 pattern.
turn 1 — 1 sub-agent
Claude main
↓ Agent call
Generator (sub-agent)
turn 2 — 2 sub-agenți în paralel
Claude main
↓ 2 Agent calls în același message
Control (sub-agent)
Conservator (sub-agent)
✓ Independență reală — sub-agenții returnează doar JSON final, fără reasoning comun.
Use when: decizii subtile unde contaminarea contează, high-stakes (migrări DB, security, refactor mare), sau când user-ul cere explicit „in paralel”.
Max 3 sub-agenți activi simultan; în practică 1 + 2 pentru că Control și Conservator depind de output-ul Generator-ului. Prompts-urile prompts/*.md sunt deja self-contained — pot fi date direct unui sub-agent fără ajustări.
